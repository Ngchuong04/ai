# Meta-Skills

Skills that orchestrate other skills, coordinate multi-step workflows, and provide higher-order development patterns. These meta-skills don't implement domain logic directly — they route to and compose specialized skills for end-to-end results.

## Skills

| Skill | Description |
|-------|-------------|
| [10x-patterns](10x-patterns/) | Patterns and practices that dramatically accelerate development velocity |
| [ai-chat-application](ai-chat-application/) | Complete guide to building AI chat applications with Vercel AI SDK |
| [auto-context](auto-context/) | Automatically read relevant project context before major actions |
| [context-driven-development](context-driven-development/) | Treat project context as a managed artifact alongside code using structured documents |
| [decision-frameworks](decision-frameworks/) | Structured decision-making patterns for common engineering choices |
| [design-system-creation](design-system-creation/) | Complete workflow for creating distinctive design systems from scratch |
| [estimation-patterns](estimation-patterns/) | Practical estimation techniques for software tasks and sprint planning |
| [feature-specification](feature-specification/) | Convert persona docs into detailed feature specifications with acceptance criteria |
| [full-stack-feature](full-stack-feature/) | Orchestrate end-to-end feature development from persona research through deployed code |
| [planning-with-files](planning-with-files/) | File-based planning using persistent markdown files as working memory for complex tasks |
| [production-readiness](production-readiness/) | Orchestrate logging, monitoring, security, and testing skills for launch readiness |
| [project-documentation](project-documentation/) | Complete workflow for project documentation including ADRs, PRDs, and docs organization |
| [realtime-dashboard](realtime-dashboard/) | Complete guide to building real-time dashboards with streaming data and live updates |
| [security-review](security-review/) | Orchestrate comprehensive security reviews across auth, validation, secrets, and infrastructure |
| [subagent-development](subagent-development/) | Execute implementation plans by dispatching fresh subagents per task with two-stage review |
| [workflow-patterns](workflow-patterns/) | Systematic task implementation using TDD, phase checkpoints, and structured commits |
